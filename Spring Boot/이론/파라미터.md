# 파라미터

## 파라미터란?

메소드가 외부로부터 **입력 값을 전달 받기 위해 사용하는 변수**

## 예시) 커피 머신 비유

- **파라미터가 없는 경우**
    
    ```java
    String makeCoffe(){
        return "아메리카노";
    }
    ```
    
    해당 메소드는 항상 똑같은 결과물, 즉 **아메리카노**만 출력한다.
    

- **파라미터가 있는 경우**
    
    ```java
    String makeCoffe(String beenType){
        return beenType + " 원두로 내린 커피;
    }
    ```
    
    해당 메소드는 `beanType`이라는 **파라미터 (재료)**를 받는다.
    
    이 파라미터를 통해 어떤 원두를 넣을지 알려줄 수 있다.
    
    - `makeCoffe(”케냐”)` 호출 → 케냐 원두로 내린 커피
    - `makeCoffe(”에티오피아”)` 호출 → 에티오피아 원두로 내린 커피

이처럼 **파라미터**는 함수가 고정된 작업만 하는 것이 아니라, **외부에서 주어진 값에 따라 유연하게 다른 결과를 만들 수 있도록** 해주는 핵심적인 역할을 한다.

## 파라미터 vs 아규먼트

코딩에서는 두 용어를 구분해서 쓰기도 한다.

- **파라미터 (Parameter)** :
    - **함수의 정의 부분에 있는 변수**
        
        즉, 어떤 종류의 재료가 들어올지 알려주는 **‘재료 투입구’** 역할을 한다. (`String beenType`)
        
- **아규먼트 (Argument)** :
    - **함수를 호출할 때 실제로 전달하는 값.**
        
        즉, **‘실제 재료’** 역할을 한다. (`”케냐”`, `“에티오피아”`)
